Welcome to Sass

Sass is a CSS pre-compiler that provides you with great uses.

What is Sass?

Sass - Syntactically Awesome Style Sheets - is a open-source preprocessor scripting language that is either interpreted or compiled into Cascading Style Sheets (CSS). While Sass is a pre-compiler, SassScript is the scripting language. Sass is an extension to Vanilla CSS, and fully compatible with all versions of CSS. It was designed by Hampton Catlin and developed by Natalie Weizenbaum in 2006. Before you learn this language, you should at least have a basic understanding of HTML and CSS though.

How to use Sass?

As mentioned, unless Sass is compiled into Vanilla CSS, a browser cannot read it. There are several ways of compiling this language, but three of them are by far the most popular and efficient.

  • 1. Using NPM(Node Package Manager); you first download the Node JS application from NodeJS.org . It is complicated enough for most beginners, so watch this youtube video.
  • 2. By installing 'Koala GUI App'. It is a visual application, so it is really easy to compile Sass with this app. Download it at koala-app.com
  • 3. If you are using VS Code, I definitely recommend using its extension named Live Sass Compiler by Retwick Dey, who is also a developer of Live Server. I personally create Sass projects with it, so I guarantee this extension is worth using.

Its Syntax

Sass pre-compiler is divided into two types of file extensions in terms of its syntax: style.sass and style.scss . The former uses indentation to write code blocks like Python, whereas the latter looks similar to CSS and uses curly brackets and semi-colons(That's why its full name is SassyCSS!). Both may be best for different developers depending on their preferance for indents or brackets. However, their functionality is the same.

Welcome to Bootstrap!

Bootstrap is by far the most popular CSS framework, based on Sass.

What is Bootstrap?

Bootstrap is an open-source framework that is designed to speed up front-end development. Since it uses pre-made HTML and CSS templates and classes, there is no need to build everything from scratch, which is true to Vanilla CSS. Accordingly, using Bootstrap makes it not only fast, but also much, much easier to create front-end UIs and layouts. This framework also includes several JS plugins: modals, collapses, accordions, and some more. This little collapse module is also created with Bootstrap Collapse Module! Another reason for its popularity is that it uses an efficient way of making UIs responsive, that is, mobile-first.

How to use Bootstrap?

Bootstrap is a third-party library, so it should be included in your page in one of the thre following ways:

  • 1. Include the CDN (Content Delivery Network) at cdnjs.com. It is the most reliable way and fastest way of using Bootstrap and its classes. If you want to use JS plugins as well, you should include bootstrap JS script at the bottom of your body tag. It is definitely recommended unless you want to customize Bootstrap - its colors, widths, grid-system, and other classes.
  • 2. Download & Include compiled Sass and JS Files at bootstrap.com. If you are not using Bootstrap for very big projects, you need to customize your layouts, then i recommend using Bootsrap this way. It has become to make changes to Bootstrap 5.0.2 as it released its Utilities API.. This video by Traversy Media shows how to achieve this.
  • 3. Use NPM (Node Package Manager). It is similar to that of Sass.

Why use them both?

Reasons for the use of Sass and Bootstrap

Advantages of Sass

There are many advantages in using this language; once you get the grasp of Sass, I have every reason to believe, you probably won't go back to Vanilla CSS.

  • Sass contains nesting just like that in HTML. If nested, it becomes much easier to locate, change, and work with styles.
  • Styles written in Sass are compiled into several browser formats( excluding IE ). This means that this framework is far more compatible with browsers compared to pure CSS. Although you can do this in your file, it can become tiring.
  • Sass includes a super way of building a project tree. It includes partials, and this enables you to separate different styles into different files. For example, one file for buttons, another for tables, yet another for logins. It makes your code much clean and efficient.
  • As said before, Sass is like a dynamic language, that is, there are functions, mix-ins, loops, built-in functions for a variety of components; lighten($primary-color, 50%) is one to illustrate. There is little need, I hope, to explain their benefits for us.

Why Bootstrap?

A quick fact: Bootstrap's styles are built on Sass, and can be customized by Sass. Bootstrap is very efficient and fast. It is especially handy for those that lack design skills and deal mainly with backend development.

  • Bootstrap is a set of pre-defined or utility classes ; you don't create anything from scratch. That means building faster with little involvement in CSS.
  • A good and experienced designer can easily make layouts very responsive as well as user experience amazing. The same can be said about one in Bootstrap. You don't have to worry much about responsiveness, which is a big plus.
  • Bootstrap's design is satisfactory though not the best in the world. If your team or you cannot afford a designer, Bootstrap is a good choice for that.
  • A big jump from a static website to a dynamic one offers Bootstrap. It includes several JS plugins: accordion, dropdown, modals, carousel, tooltips, and some more. It is not hard to implement them, thereby giving your page a dynamic state.

So, what then?

It is totally up to your preferances what to use. But given the advantages, the two frameworks can boost your efficiency and it is good considering this.
